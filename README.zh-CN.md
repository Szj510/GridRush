# GridRush：九宫格竞速

[English](README.md)

**GridRush** 是一款竞争激烈的实时多人网页游戏，玩家通过完成快节奏的小游戏来争夺 3×3 九宫格的控制权。

![状态](https://img.shields.io/badge/状态-Alpha-blue) ![许可证](https://img.shields.io/badge/许可证-CC%20BY--NC%204.0-lightgrey) ![React](https://img.shields.io/badge/React-19-blue)

## 🎮 游戏特色

- **在线对战** — 基于 WebRTC（PeerJS P2P）的实时 1v1 对战，无需服务器。
- **可靠中继模式** — 当配置了 Supabase 环境变量后，在线消息默认走 Supabase Realtime 中继，避免纯 P2P 打洞不稳定问题。
- **机器人引导赛** — 第一次进入在线对战时，可以启动一场脚本化新手对局，边打边学会占格、冻结和抢夺。
- **链接邀请** — 创建房间后点击 **复制邀请链接**，将链接发给好友——对方打开后直接进入你的房间，无需手动输入房间号。
- **竞速模式** — 单人按顺序尽快通关全部 13 个小游戏。
- **练习道场** — 针对单个小游戏进行专项练习，支持自定义难度。
- **赛前禁选** — 进入棋盘前，双方各禁选 1 个小游戏，被禁选项目不会进入本局九宫格图池。
- **技能系统** — 每局在线对战开始前，从 3 种技能中秘密选择 2 种，对手无法得知你的选择！
  - ⭐ **夺旗** — 挑战对手已占领的格子，对手有 5 秒时间防守，否则格子归你。
  - ❄️ **冻结** — 冻结对手的操作 2 秒。
  - ⚔️ **决斗** — 强制双方同时在你选定的空格竞速，更快者占领该格。
- **娱乐模式卡牌** — 占领格子后可抽取随机道具卡（共 10 种）：黑幕、混牌、弹射、炼狱、翻转、炸弹、换牌、吸附、冰冻、置换。
- **数据统计面板** — 可视化展示近期与生涯数据：在线胜负平、动作累计、猜拳胜率、竞速最佳和最近对局记录。
- **成就系统** — 基于本地存储的成就记录，跨会话持久保存。
- **双语支持** — 完整的中文与英文界面支持。
- **深色模式** — 支持亮/暗主题切换。

## ⚡ 首局 30 秒上手

1. 在线模式开局时，双方会先各禁选 1 个小游戏。
2. 然后双方再从抢夺、冻结、决斗这 3 个技能里秘密选 2 个。
3. 接着进行三局两胜的猜拳，输的一方在棋盘开局时会先被冻结 2 秒。
4. 点击任意带“?”的格子进入对应小游戏，先通关的人就立刻占领该格。
5. 进入对手已占的格子会消耗你的抢夺，并给对手 5 秒的防守响应时间。
6. 连成三格立即获胜；如果 9 格全部占满，则占格更多者获胜。

如果你是第一次接触在线模式，建议先从在线大厅里的机器人引导赛开始。它会在真实对局里一步步提示你该怎么打。

### 需要特别注意的惩罚

- 同一格连续失败 3 次，会被该格禁入 10 秒。
- 抢夺途中主动放弃，会触发 20 秒抢夺冷却。
- 长时间无操作或断线太久，会被系统踢出当前正在争夺的格子。

## 🛠️ 技术栈

- **React 19** — 前端 UI 库
- **TailwindCSS**（CDN 版）— 样式与动画
- **PeerJS** — 用于 P2P 联网的 WebRTC 封装库
- **Vite** — 构建工具与开发服务器
- **Supabase**（可选）— 账号系统与云端统计同步（需配置 `.env.local`）

## 🌐 联机稳定性

- `VITE_NET_USE_SUPABASE_RELAY=1`（当存在 Supabase 配置时默认开启）会让对局消息走 Supabase Realtime 中继。
- `VITE_NET_USE_SUPABASE_RELAY=0` 可强制回退到旧版 PeerJS P2P 模式。
- 如果你仍使用 PeerJS 且网络无法访问 `0.peerjs.com`，请在 `.env.local` 中配置 `VITE_PEERJS_HOST` 等自建信令参数。

## 🔒 安全设计

GridRush 完全运行在浏览器中，无后端服务器。所有游戏状态存储在**房主**端，客机只发送动作消息，由房主权威处理。所有来自对端或本地存储的数据均被视为不可信输入：

| 防护层                  | 措施                                                                                                                                       |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------ |
| **P2P 消息（房主侧）**  | 每条收到的 `NetworkMessage` 都由 `services/sanitize.ts` 从原始数据重新构建——类型字段对照白名单校验，数值字段范围限制。                     |
| **P2P 消息（客机侧）**  | 来自房主的 `STATE_UPDATE` 通过形状验证（状态白名单、格子数量上限、必要字段存在性检查）后才应用到本地状态。                                 |
| **禁选注入**            | 客机发送的禁选小游戏 ID 只接受已知小游戏列表中的值，且每位玩家每回合只接受一次。                                                           |
| **技能选择注入**        | 客机发送的技能 ID 过滤至固定白名单（`STEAL`、`FREEZE`、`DUEL`），无法注入任意字符串。                                                      |
| **动作洪泛限制**        | `RateLimiter`（令牌桶）在房主侧执行每动作速率限制：`HEARTBEAT` ≤ 3/s，`CLICK_CELL` ≤ 10/s，`INTERACTION` ≤ 20/s，`COMPLETE_GAME` ≤ 2/2 s。 |
| **房间号注入**          | `joinGame` 将用户输入与 `/^\d{4}$/` 正则匹配校验后再传入 PeerJS，防止构造任意 Peer ID。                                                    |
| **本地存储污染**        | `sanitizeSettings` / `sanitizeStats` 逐字段重建对象并做类型校验或范围限制，而非直接使用 JSON 解析结果。                                    |
| **内容安全策略（CSP）** | `<meta>` CSP 标签限制脚本来源白名单，将 WebSocket/XHR 锁定到 `*.peerjs.com`，并阻止 `frame-src` / `object-src` 以防嵌入攻击。              |

> **固有局限**：缺乏可信服务器时，房主状态具有权威性但无法被客机验证。作弊的房主仍可发送伪造的 `STATE_UPDATE`。上述措施用于防范**客机作弊**以及防止本地浏览器数据被意外或恶意篡改。

## 🚀 本地运行

**前置要求：** 安装 [Node.js](https://nodejs.org/) v20 或更高版本。

```bash
# 克隆仓库
git clone https://github.com/Szj510/grid-rush.git
cd grid-rush

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

用浏览器打开 `http://localhost:5173`。如需模拟玩家 2，在另一个标签页打开同一地址即可。

### 构建发布版本

```bash
npm run build
# 构建产物位于 dist/ 目录
```
